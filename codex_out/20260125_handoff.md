# 2026-01-25 ハンドオフ（カテゴリ同期／マッチング）

## これまでの理解
- 目的: Amazon BTG（AmaCategory）と Wowma カテゴリ（WowCategory）を最新化し、カテゴリ対応を自動/半自動で決めるための仕組み作り。Wowmaの商品登録時に Amazon ASIN のカテゴリから Wowma カテゴリを引けるようにする。
- データ元:
  - Amazon BTG: `category/amazon/*.xls`（シート2に Node ID / Node Path）
  - Wowmaカテゴリ: `category/wowma/category.xlsx`
  - 第1階層対応表: `docs/category_mapping/level_1_cat.txt`（Ama第1階層 → Wow第1階層の許可リスト）
  - レビュー用出力: `docs/category_mapping/category_match_review.tsv`, `category_match_review_level2.tsv`
- モデル: `AmaCategory`（level_1〜8, wow_cat_id 等）、`WowCategory`（level_1〜4, ama_level_*）

## 実装・変更点
- 依存追加: `xlrd==1.2.0`, `openpyxl==3.1.2`（Excel読み取り用）
- 管理コマンド追加:
  - `import_categories`: BTG/Wowma を Excel から読み込み、truncate + bulk_create。
  - `match_categories`: 第1階層マッピングを必須フィルタとして候補絞り込み。synonyms 正規化＋fuzzy で auto/review/review_level2 TSV を出力。  
    オプション: `--synonyms`, `--level1-map`, `--auto-threshold`, `--review-threshold`, `--outdir`
- ファイル:
  - `docs/category_mapping/level_1_cat.txt` … Ama第1階層と許可する Wow第1階層の組み合わせ。
  - `docs/category_mapping/synonyms.tsv` … AmaPath から抽出した英単語をすべて展開し、英→英の素振りは完了。**日本語訳の付与は未完成（次課題）。**
  - `docs/category_mapping/review_format.tsv` … TSV ヘッダー定義。

## 現状・結果
- import: AmaCategory 21,446 件 / WowCategory 7,874 件に更新（VPSで実行済み）。
- match結果（最新実行時点）: auto=0, review_level2=215, review=3,304。synonymsの日本語不足のため自動採用がゼロ。第1階層フィルタは適用済み。
- synonyms.tsv は UTF-8 で再生成済みだが、日本語訳が英語のみの行多数 → 要整備。

## 次に必要なこと（引き継ぎ）
1. **synonyms.tsv の日本語拡充**  
   - `docs/category_mapping/category_match_review.tsv` と `..._level2.tsv` の AmaPath から抽出した英単語 2,300語超を、それぞれ適切な日本語同義語に置き換え/追加する。現状は英語素振りだけなのでマッチ精度が低い。
2. **再マッチ**  
   - `python manage.py match_categories --synonyms docs/category_mapping/synonyms.tsv --level1-map docs/category_mapping/level_1_cat.txt --outdir codex_out --auto-threshold 0.7 --review-threshold 0.4`
   - 依存は `pip install -r requirements.txt` 済みであれば不要。
3. **レビュー/反映**  
   - `codex_out/category_match_auto.tsv`/`..._review.tsv`/`..._review_level2.tsv` を確認。レビュー結果で確定した対応を DB へ反映する更新コマンド（未実装）を追加・実行する。
4. **安全手順**  
   - 本番 DB は mysqldump でバックアップを取ってから反映。
   - 第1階層の対応表 `level_1_cat.txt` にない組み合わせは候補に入らない運用。

## 実行手順（再掲）
```bash
# venv有効化済み前提
python manage.py import_categories --source amazon --truncate
python manage.py import_categories --source wowma --truncate
python manage.py match_categories \
  --synonyms docs/category_mapping/synonyms.tsv \
  --level1-map docs/category_mapping/level_1_cat.txt \
  --outdir codex_out \
  --auto-threshold 0.7 --review-threshold 0.4
```

## 既知の課題
- synonyms.tsv の日本語訳が未整備 → 自動マッチがゼロ。
- 自動採用閾値を下げる検討（例 0.6）もあるが、誤マッチリスクが高いため synonyms 整備が優先。
